# Phase 1 Intake Prompt

You are preparing a Phase 1 plan-space workflow. Produce a structured context packet.

## Output Format (use these exact headings)

### Problem Statement
- One paragraph describing the problem and why it matters now.

### Target Users
- Primary user personas and their goals.

### Success Criteria
- 3 to 7 measurable outcomes that define success.

### Scope and Non-Goals
- MVP scope (must-have capabilities)
- Explicit non-goals

### Constraints
- Timeline
- Budget
- Security/compliance
- Integrations/dependencies

### Existing Assets
- Current codebase, APIs, data sources, or infrastructure

### Open Questions
- List unknowns that must be answered before execution

### Assumptions Ledger
- Each assumption with status: validated, plausible, or unknown/unspecified

## Guardrails
- If any input is missing, mark it as unknown/unspecified rather than guessing.
- Keep this concise and self-contained for a future reader.
- Treat this as the output contract for Phase 1 inputs; do not add unstated fields.
